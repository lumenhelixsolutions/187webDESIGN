<h2 id="-what-this-is">
  <img src="https://readme-typing-svg.demolab.com?font=Space+Grotesk&weight=700&size=26&pause=1000&color=2440E6&center=false&vCenter=true&width=420&height=40&lines=%E2%97%88+What+this+is" alt="What this is" />
</h2>

This repository is **two things in one**:

1. **A reusable skill** — [`/.claude/skills/187webdesign/`](./.claude/skills/187webdesign/SKILL.md) is a portable, tool-agnostic design playbook for [Claude Code](https://claude.com/claude-code). Drop it into any project and Claude designs and critiques web pages against an award-winning rubric, a 12-point principle set, and a 50-point pre-ship audit.
2. **A full-stack starter that demonstrates it** — a real Next.js + Prisma app whose marketing page is built to pass that exact checklist: fast (Core Web Vitals), accessible (WCAG 2.2 AA), responsive, and conversion-minded. Clone it and start shipping, or read it as a worked example.

<h2 id="-the-scoring-lens">
  <img src="https://readme-typing-svg.demolab.com?font=Space+Grotesk&weight=700&size=26&pause=1000&color=2440E6&center=false&vCenter=true&width=420&height=40&lines=%E2%97%88+The+scoring+lens" alt="The scoring lens" />
</h2>

How "award-winning" is actually judged — design to the **weights**, not just to taste. This is the rubric the skill (and this page) is built around:

```text
DESIGN       ████████████████████████████████   40%   craft · type · color · detail
USABILITY    ████████████████████████           30%   nav · speed · a11y · responsive
CREATIVITY   ████████████████                   20%   one fresh, memorable idea
CONTENT      ████████                           10%   copy that earns the click
```

> A beautiful site that's hard to use still loses. The gap between *fine* and
> *award* is **craft details + one bold idea + fast, accessible execution** —
> not more effects.

## 🛠️ Tech stack

<div align="center">

| Layer | Choice | Why |
|:--|:--|:--|
| **Framework** | Next.js 15 (App Router) | Full-stack React: UI + API in one deploy |
| **Language** | TypeScript (strict) | Catch mistakes before users do |
| **Styling** | Tailwind CSS 3 + CSS-variable tokens | Utility speed with a real token system |
| **Database** | PostgreSQL via Prisma 6 | Typed schema, easy local ↔ prod parity |
| **Validation** | Zod | One schema shared by API and client |
| **Fonts** | Space Grotesk + Inter | Two families, self-hosted via `next/font` |
| **Tooling** | ESLint · Prettier · GitHub Actions | Lint + typecheck + build on every push |

</div>

<h2 id="-quick-start">
  <img src="https://readme-typing-svg.demolab.com?font=Space+Grotesk&weight=700&size=26&pause=1000&color=2440E6&center=false&vCenter=true&width=340&height=40&lines=%E2%9A%A1+Quick+start" alt="Quick start" />
</h2>

**Prerequisites:** Node.js ≥ 18.18, npm. (Docker optional, only for the local database.)

### A) The full starter — clone &amp; build

```bash
git clone https://github.com/lumenhelixsolutions/187webDESIGN.git
cd 187webDESIGN
npm install
npm run dev          # → http://localhost:3000
```

The showcase runs immediately in **preview mode** — no database required. Wire a DB when you want to persist leads:

```bash
cp .env.example .env
docker compose up -d        # provisions Postgres matching .env
npm run db:push             # create tables from prisma/schema.prisma
npm run db:seed             # (optional) a few sample leads
```

### B) Just the skill — drop-in

```bash
# from the root of any project you use with Claude Code
cp -r path/to/187webDESIGN/.claude/skills/187webdesign .claude/skills/
```

Claude Code auto-discovers it. Then ask *"design an award-caliber hero for X"* or *"critique this landing page against the rubric."*

The heart of the repo. [`SKILL.md`](./.claude/skills/187webdesign/SKILL.md) encodes:

- **The scoring lens** — how awards are judged: Design 40% · Usability 30% · Creativity 20% · Content 10%.
- **12 core principles** — from *"win the first 5 seconds"* to *"convert, don't just impress."*
- **A workflow** — research → direction → wireframe → tokens → build → critique → QA.
- **A 50-point pre-ship checklist** — [`CHECKLIST.md`](./.claude/skills/187webdesign/references/CHECKLIST.md).

It's intentionally tool-agnostic: the principles hold whether you build in React, plain HTML/CSS, Framer, or Webflow.

<br>

## 🎨 Design tokens

Defined once in [`app/globals.css`](./app/globals.css), surfaced to Tailwind in [`tailwind.config.ts`](./tailwind.config.ts). Full reference: [`docs/DESIGN-SYSTEM.md`](./docs/DESIGN-SYSTEM.md).

<div align="center">

| Token | Value | Role |
|:--|:--|:--|
| `bg` | `#FAFAF7` | Warm paper background (the 60%) |
| `ink` | `#11131A` | Primary text / dark surfaces |
| `muted` | `#5B6170` | Secondary text (AA on `bg`) |
| `accent` | `#2440E6` | Electric "blueprint" blue — actions only (the 10%) |
| `line` | `#DDDBD2` | Hairlines / blueprint rules |

## 🚀 Deployment

- **GitHub Pages (the live demo).** Pushes to `main` build a fully static export and publish it via [`.github/workflows/pages.yml`](./.github/workflows/pages.yml). One-time setup: **Settings → Pages → Source: GitHub Actions**. *(The static demo is visual-only — the live API and database run in the full-stack deploys below.)*
- **Vercel.** Import the repo, set `DATABASE_URL` (+ `NEXT_PUBLIC_SITE_URL`), deploy. Run `prisma migrate deploy` against your database.
- **Docker.** `docker build -t 187webdesign . && docker run -p 3000:3000 -e DATABASE_URL=… 187webdesign`.

Full guide: [`docs/DEPLOYMENT.md`](./docs/DEPLOYMENT.md).
